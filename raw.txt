Since you're using Anti-Gravity to generate the application, don't start with technical implementation details. Start with the business objective, workflow, screens, and constraints.

You can give Anti-Gravity a prompt like this:

Project Name: Guardian Voice Agent POC

Objective:
Build a working Proof of Concept (POC) for a hospital voice agent that can communicate with patients regarding appointments and update appointment status based on the conversation.

Important Constraints:

This is a 1-day POC.
Focus on proving the voice agent workflow.
Do not build enterprise features.
Do not implement authentication.
Do not implement a database.
Use local mock data stored in TypeScript or JSON.
The application should be easy to extend later.

Core Workflow:

Dashboard displays appointments.
User selects an appointment.
User clicks "Start Browser Call".
Guardian Voice Agent starts a voice conversation.
Agent asks:
"Would you like to confirm, cancel, or reschedule your appointment?"
Patient responds by voice.
AI detects intent:
Confirm
Cancel
Reschedule
Dashboard updates appointment status.
Activity log records the interaction.
Doctor notification is generated and displayed.

Voice Agent Requirements:

Browser-based voice conversation.
Use microphone input.
Use speech-to-text.
Use an LLM to classify intent.
Use text-to-speech for responses.
No telephony required for the main workflow.

Future Support:

Architecture should allow future integration with:
Phone calls
Twilio
WhatsApp notifications
SMS notifications
Real scheduling systems
Hospital databases
